# Roadmap и история проекта Avitolog

Документ фиксирует, **что было** в репозитории до крупного упрощения и **что изменилось** после него, чтобы сохранялась понятная история для команды и для будущих правок.

## Текущее состояние (после упрощения)

**Цель продукта:** один Telegram-бот: после `/start` бот просит **только текст объявления**; это сообщение уходит в **одного** OpenAI Assistant, ответ приходит в чат (формат задаётся инструкциями ассистента в OpenAI).

**Стек:** Python 3.11+, `aiogram`, `openai`, `python-dotenv`.

**Запуск:** `python -m app.main` (нужны `BOT_TOKEN`, `OPENAI_API_KEY`).

**Конфигурация ассистента:** по умолчанию в коде задан ID `asst_ochis2TU72Qcdg5bEEHYfAMk`; при необходимости переопределяется переменной окружения `ASSISTANT_ID`.

**Структура кода:**

| Файл | Назначение |
|------|------------|
| `app/main.py` | Точка входа, polling |
| `app/config.py` | Настройки из `.env`, дефолтный `ASSISTANT_ID` |
| `app/bot/factory.py` | Сборка `Bot` + `Dispatcher`, внедрение зависимостей |
| `app/bot/handlers.py` | `/start`, обработка текста, разбиение длинного ответа |
| `app/services/openai_service.py` | Вызов Assistant API (thread + run), ретраи |

**Зависимости:** см. `requirements.txt` (без pandas/openpyxl).

**Если в Telegram снова «Шаг 1/6» и Excel:** у вас запущен **старый деплой** или другая копия проекта. Остановите процесс, убедитесь что на сервере/ПК лежит актуальный код из этого репозитория, выполните `git pull` при необходимости и **перезапустите** `python -m app.main`. В логах при старте должна быть строка: `режим «текст объявления → ответ ассистента» (без шагов и Excel)`.

## Что было раньше (до упрощения)

Ниже — описание **логики и артефактов**, которые существовали в проекте до перехода к «только один ассистент, только текст».

### Продукт и сценарий

- Telegram-бот для **авитологов**: пошаговый сценарий (FSM) — ниша → сегменты JTBD → количество объявлений → города → тип GEO → подтверждение → **генерация пачки объявлений** и выдача **Excel**.
- Упор на объявления под **Avito**: CTR, лиды, оговорки про модерацию (в т.ч. в отдельных промптах/спеках).

### Архитектура

- **Несколько агентов** (`app/agents/`): парсер ввода, JTBD-сегментация, копирайт, HTML.
- **Оркестратор** (`app/services/orchestrator.py`) собирал объявления из конфигурации (ниша, сегменты, города, GEO, число объявлений).
- **OpenAI:** чат/Responses + опционально до **трёх** Assistants по переменным `ASSISTANT_JTBD_PARSER_ID`, `ASSISTANT_CREATIVE_BUILDER_ID`, `ASSISTANT_SPINTAX_HTML_ID`; при отсутствии ID — **локальные промпты** в `app/prompts/`.
- **Сервисы:** экспорт в Excel (`excel_service`), генерация гео-фраз (`geo_service`).
- **Утилиты:** загрузка промптов, спинтакс, работа с текстом (`app/utils/`).
- **Состояния диалога:** `app/bot/states.py` (несколько шагов мастера).

### Документация и спеки (удалены при упрощении)

В корне лежали материалы вроде:

- `assistants-setup.md` — маппинг env → ассистенты.
- `assistant-jtbd-parser.md`, `assistant-creative-builder.md`, `assistant-spintax-html.md` — описание ролей ассистентов.
- `avito-conversion-architect-spec.md`, `jtbd-audience-analysis.md`, `spintax-html-one-line-spec.md` — методология и требования к текстам/спинтаксу/HTML.

Их **намеренно убрали** из репозитория вместе с многошаговым пайплайном; при необходимости историю можно восстановить из **git history**, если файлы были закоммичены.

### Зависимости (раньше)

- Дополнительно: `pandas`, `openpyxl` (Excel).

## Что сделали при упрощении (добавили / изменили)

### Добавлено

- **Один сценарий «текст → ответ ассистента»** без FSM и без Excel.
- **Явный дефолтный Assistant ID** в `app/config.py` (`DEFAULT_ASSISTANT_ID`).
- **Сообщение «Генерирую…»** и обработка ошибок с редактированием статуса.
- **Разбиение длинного ответа** на части под лимит Telegram (~4000 символов).
- **Обработка не-текстовых сообщений** с просьбой прислать текст.
- Переменная окружения **`ASSISTANT_ID`** (опционально) в `.env.example`.
- Увеличенный таймаут ожидания run ассистента (до **180** с) в `openai_service` по сравнению с прошлой логикой (раньше было 120 с в одном из путей).
- **Этот файл `ROADMAP.md`** — фиксация истории «было / стало».

### Изменено

- **`app/config.py`:** только `bot_token`, `openai_api_key`, `assistant_id`, `max_retries`; убраны модель, температура и три разных ID ассистентов.
- **`app/services/openai_service.py`:** только вызов Assistant (`run_assistant` / внутренний `_run_assistant_once`); убраны общий `generate_text`/`generate_json`, **нет fallback** на Responses API при сбое ассистента (ошибка уходит пользователю после ретраев).
- **`app/bot/factory.py`:** регистрируются только `openai_service` и `assistant_id`.
- **`app/bot/handlers.py`:** полностью новая логика вместо многошагового мастера.
- **`app/main.py`:** проверка наличия `ASSISTANT_ID` (после загрузки из env с учётом дефолта в коде).
- **`requirements.txt`:** убраны `pandas`, `openpyxl`.

### Удалено

- Каталог **`app/agents/`** (все агенты).
- **`app/services/orchestrator.py`**, **`excel_service.py`**, **`geo_service.py`**.
- **`app/bot/states.py`**, весь FSM-поток в старых handlers.
- **`app/prompts/`** (все `.txt`).
- **`app/utils/`** (prompt_loader, spintax_utils, text_utils).
- **`app/tests/generate_test.py`**.
- Перечисленные выше **`.md`** в корне репозитория.

### Без изменений по смыслу деплоя

- **`Dockerfile`**, **`Procfile`**, **`.gitignore`** — по-прежнему подходят к текущему `python -m app.main`.
